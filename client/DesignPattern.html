Design Patterns trong Lập Trình

🏗️ 1. Creational Patterns — Tạo đối tượng
Creational

Singleton

Đảm bảo chỉ có một instance duy nhất của class tồn tại trong toàn bộ ứng dụng.

Dùng khi
Database connection Logger Config manager Room DB 
object DatabaseManager {
   fun connect() { /* ... */ }
   fun query(sql: String) { /* ... */ }
}
Creational

Builder

Xây dựng object phức tạp theo từng bước, tách rời quá trình tạo khỏi representation.

Dùng khi
AlertDialog OkHttpClient Retrofit Notification 
val dialog = AlertDialog.Builder(context)
   .setTitle("Tiêu đề")
   .setMessage("Nội dung")
   .setPositiveButton("OK") { _, _ -> }
   .build()
Creational

Factory Method

Định nghĩa interface tạo object, để subclass quyết định class nào được tạo.

Dùng khi
ViewModel Factory Cross-platform UI Plugin system 
abstract class Dialog {
   abstract fun createButton(): Button
}
class WindowsDialog : Dialog() {
   override fun createButton() = WindowsButton()
}
Creational

Abstract Factory

Tạo ra họ các object liên quan mà không chỉ định class cụ thể.

Dùng khi
Dark/Light theme Cross-platform UI component family 
interface UIFactory {
   fun createButton(): Button
   fun createCheckbox(): Checkbox
}
class DarkThemeFactory : UIFactory { /*...*/ }
Creational

Prototype

Clone object hiện tại thay vì tạo mới, hữu ích khi khởi tạo object tốn chi phí cao.

Dùng khi
Copy objects Game entities Config templates 
data class UserConfig(
   val theme: String,
   val locale: String
)
val copy = config.copy(theme = "dark")
🔗 2. Structural Patterns — Cấu trúc
Structural

Adapter

Chuyển đổi interface của class sang interface khác mà client mong đợi — như bộ chuyển đổi phích cắm.

Dùng khi
RecyclerView.Adapter 3rd party lib Legacy code 
class ItemAdapter(
   private val items: List<Item>
) : RecyclerView.Adapter<ViewHolder>() {
   override fun getItemCount() = items.size
}
Structural

Decorator

Thêm behavior mới vào object tại runtime bằng cách wrap nó, không thay đổi class gốc.

Dùng khi
Java I/O Streams Middleware Logging wrapper 
val stream = BufferedInputStream(
   FileInputStream("data.txt")
)
// BufferedInputStream "decorates" FileInputStream
Structural

Facade

Cung cấp interface đơn giản cho một hệ thống con phức tạp — che giấu sự phức tạp bên trong.

Dùng khi
Repository SDK wrapper Service layer 
class MediaFacade {
   private val player = MediaPlayer()
   private val buffer = BufferManager()
   fun play(url: String) {
       buffer.prepare(url); player.start()
   }
}
Structural

Proxy

Cung cấp đối tượng thay thế để kiểm soát truy cập — lazy loading, caching, access control.

Dùng khi
Retrofit interface Lazy loading Auth guard Cache proxy 
interface ApiService {
   suspend fun getUsers(): List<User>
}
// Retrofit tạo Proxy implement ApiService
val api = retrofit.create(ApiService::class.java)
Structural

Composite

Xử lý object đơn lẻ và nhóm object theo cách thống nhất — cấu trúc cây.

Dùng khi
View hierarchy File system Menu tree 
// Android: ViewGroup là Composite của View
val layout = LinearLayout(context)
layout.addView(TextView(context))
layout.addView(ImageView(context))
// draw() gọi đệ quy xuống toàn bộ cây
Structural

Bridge

Tách abstraction khỏi implementation để cả hai có thể thay đổi độc lập nhau.

Dùng khi
Multi-platform renderer Device drivers Remote/Local data 
interface Renderer { fun render(shape: Shape) }
class OpenGLRenderer : Renderer { /*...*/ }
class VulkanRenderer  : Renderer { /*...*/ }
class Circle(val renderer: Renderer) : Shape()
⚡ 3. Behavioral Patterns — Hành vi
Behavioral

Observer

Một object thay đổi → thông báo tự động đến tất cả các object đang theo dõi nó.

Dùng khi
LiveData StateFlow RxJava EventBus 
viewModel.uiState
   .collect { state ->
       updateUI(state)
   }
// StateFlow là Observer pattern
Behavioral

Strategy

Định nghĩa một họ thuật toán, đóng gói từng cái, cho phép hoán đổi linh hoạt tại runtime.

Dùng khi
Payment methods Sort algorithms Compression 
interface SortStrategy {
   fun sort(data: List<Int>): List<Int>
}
class Sorter(var strategy: SortStrategy) {
   fun sort(d: List<Int>) = strategy.sort(d)
}
Behavioral

Command

Đóng gói request như một object — hỗ trợ undo/redo, queue, logging các action.

Dùng khi
Undo/Redo Transaction Macro recording Task queue 
interface Command {
   fun execute()
   fun undo()
}
class DeleteTextCommand : Command {
   override fun execute() { /* xóa */ }
   override fun undo() { /* khôi phục */ }
}
Behavioral

Chain of Responsibility

Truyền request qua chuỗi handler cho đến khi có handler xử lý được.

Dùng khi
OkHttp Interceptor Touch events Middleware 
// OkHttp Interceptor là Chain of Responsibility
client = OkHttpClient.Builder()
   .addInterceptor(AuthInterceptor())
   .addInterceptor(LoggingInterceptor())
   .addInterceptor(CacheInterceptor())
   .build()
Behavioral

Template Method

Định nghĩa skeleton của thuật toán trong base class, để subclass override các bước cụ thể.

Dùng khi
Activity lifecycle Data export Report generator 
abstract class DataExporter {
   fun export() { read(); process(); write() }
   abstract fun read()
   abstract fun write()
   open fun process() { /* default impl */ }
}
Behavioral

State

Object thay đổi behavior dựa trên state nội tại — như máy trạng thái (FSM).

Dùng khi
Media player UI State Machine Order status 
sealed class UiState {
   object Loading : UiState()
   data class Success(val data: List<Item>) : UiState()
   data class Error(val msg: String) : UiState()
}
Behavioral

Iterator

Cung cấp cách tuần tự truy cập các phần tử mà không expose cấu trúc bên trong.

Dùng khi
for-each Kotlin Iterable Custom collection 
class NumberRange(val from: Int, val to: Int) : Iterable<Int> {
   override fun iterator() = (from..to).iterator()
}
for (n in NumberRange(1, 5)) println(n)
Behavioral

Mediator

Giảm sự phụ thuộc trực tiếp giữa các object bằng cách giao tiếp qua một đối tượng trung gian.

Dùng khi
Chat room Air traffic control EventBus 
class EventBus {
   private val listeners = mutableMapOf<String, MutableList<() -> Unit>>()
   fun emit(event: String) = listeners[event]?.forEach { it() }
}

💡 Lời khuyên thực tế

Không dùng pattern vì pattern

Chỉ áp dụng khi nó thực sự giải quyết được vấn đề cụ thể. Over-engineering là kẻ thù của clean code.

Android hiện đại

Observer (Flow/StateFlow), Repository (Facade+Proxy), Factory (Hilt/DI) là phổ biến nhất trong MVVM.

MVVM = Tổ hợp patterns

MVVM bản thân là tổ hợp của Observer + Facade + Strategy — patterns không tồn tại độc lập.

Học từ thư viện

OkHttp, Retrofit, Room, Coroutines — đều là những ví dụ xuất sắc về cách áp dụng design patterns trong thực tế.
